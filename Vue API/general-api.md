# Vue General API

> Источник: [https://vuejs.org/api/general](https://vuejs.org/api/general)
>
> **XLSX:** [`general-api.xlsx`](./general-api.xlsx) (генерация: `py -3 scripts/generate-general-api-xlsx.py`)

| API | Описание | По умолчанию | Пример | Детали | Ссылки |
|-----|----------|--------------|--------|--------|--------|
| `version` | Возвращает текущую версию Vue. | — | <pre><code>import { version } from 'vue'<br><br>console.log(version)</code></pre> | Тип: `string`. Полезно для проверки версии при установке плагинов или условной логики. | — |
| `nextTick()` | Утилита для ожидания следующего цикла обновления DOM. | — | <pre><code>import { ref, nextTick } from 'vue'<br><br>const count = ref(0)<br><br>async function increment() {<br>  count.value++<br>  // DOM ещё не обновлён<br>  console.log(document.getElementById('counter').textContent)<br><br>  await nextTick()<br>  // DOM обновлён<br>  console.log(document.getElementById('counter').textContent)<br>}</code></pre> | После изменения реактивного состояния Vue буферизует обновления DOM до «next tick». Можно передать callback или await Promise. | [Component Instance — $nextTick](https://vuejs.org/api/component-instance#nexttick) |
| `defineComponent()` | Type helper для определения компонента Vue с выводом типов. | — | <pre><code>import { defineComponent } from 'vue'<br><br>const Foo = defineComponent({<br>  /* options */<br>})<br><br>type FooInstance = InstanceType&lt;typeof Foo&gt;<br><br>// function syntax (3.3+):<br>const Comp = defineComponent(<br>  (props) => {<br>    const count = ref(0)<br>    return () => h('div', count.value)<br>  },<br>  { props: { /* ... */ } }<br>)</code></pre> | Options syntax: первый аргумент — объект опций, возвращает constructor с inferred instance type. Function syntax (3.3+): функция как setup(), возвращает render function (h/JSX); поддерживает generics в TSX. Для webpack treeshaking добавьте `/*#__PURE__*/` перед вызовом (для Vite не требуется). | [Using Vue with TypeScript](https://vuejs.org/guide/typescript/overview) |
| `defineAsyncComponent()` | Определяет асинхронный компонент с ленивой загрузкой при рендере. | — | <pre><code>import { defineAsyncComponent } from 'vue'<br><br>const AsyncComp = defineAsyncComponent(<br>  () => import('./components/AsyncComponent.vue')<br>)<br><br>// с опциями:<br>defineAsyncComponent({<br>  loader: () => import('./Comp.vue'),<br>  loadingComponent: LoadingComp,<br>  errorComponent: ErrorComp,<br>  delay: 200,<br>  timeout: 3000,<br>  suspensible: false,<br>  onError(error, retry, fail, attempts) { /* ... */ }<br>})</code></pre> | Аргумент — loader-функция `() => Promise&lt;Component&gt;` или объект `AsyncComponentOptions` с полями loader, loadingComponent, errorComponent, delay, timeout, suspensible, onError. | [Async Components](https://vuejs.org/guide/components/async) |
