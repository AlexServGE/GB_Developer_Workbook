# Vue Composition API — setup()

> Источник: [https://vuejs.org/api/composition-api-setup](https://vuejs.org/api/composition-api-setup)
>
> **XLSX:** [`composition-api-setup.xlsx`](./composition-api-setup.xlsx) (генерация: `py -3 scripts/generate-composition-api-setup-xlsx.py`)

| API | Описание | По умолчанию | Пример | Детали | Ссылки |
|-----|----------|--------------|--------|--------|--------|
| `setup()` | Точка входа Composition API в компоненте. | — | <pre><code>import { ref } from 'vue'<br><br>export default {<br>  setup() {<br>    const count = ref(0)<br>    return { count }<br>  },<br>  mounted() {<br>    console.log(this.count) // 0<br>  }<br>}</code></pre> | Используется без SFC/build step или для интеграции Composition API в Options API компонент. Возвращает объект — свойства доступны в template и на this (Options API). Внутри setup() this === undefined. Refs из return автоматически unwrap в template и на this. Рекомендуется script setup в SFC. | [Composition API — setup()](https://vuejs.org/guide/essentials/composition-api-setup) |
| `setup(props)` | Первый аргумент setup — реактивные props компонента. | — | <pre><code>export default {<br>  props: { title: String },<br>  setup(props) {<br>    console.log(props.title)<br>  }<br>}</code></pre> | props реактивны и обновляются при новых значениях. Деструктуризация props теряет reactivity — обращаться как props.xxx. | [Composition API — setup()](https://vuejs.org/guide/essentials/composition-api-setup) |
| `toRefs(props)` / `toRef(props, key)` | Утилиты для деструктуризации props с сохранением reactivity. | — | <pre><code>import { toRefs, toRef } from 'vue'<br><br>setup(props) {<br>  const { title } = toRefs(props)<br>  console.log(title.value)<br><br>  const titleRef = toRef(props, 'title')<br>}</code></pre> | toRefs(props) превращает props в объект refs. toRef(props, 'title') — ref для одного свойства. Нужны при деструктуризации или передаче prop во внешнюю функцию. | [toRefs()](https://vuejs.org/api/reactivity-utilities#torefs), [toRef()](https://vuejs.org/api/reactivity-utilities#toref) |
| `async setup()` | Асинхронная форма setup с top-level await. | — | <pre><code>export default {<br>  async setup() {<br>    const data = await fetchData()<br>    return { data }<br>  }<br>}</code></pre> | Допустима только если компонент — потомок Suspense. Иначе setup() должен возвращать объект синхронно. | [Composition API — setup()](https://vuejs.org/guide/essentials/composition-api-setup) |
| `setup(props, context)` | Второй аргумент — объект Setup Context. | — | <pre><code>setup(props, context) {<br>  console.log(context.attrs)<br>  console.log(context.slots)<br>  console.log(context.emit)<br>  console.log(context.expose)<br>}</code></pre> | context не реактивен, можно безопасно деструктурировать: { attrs, slots, emit, expose }. | [Composition API — setup()](https://vuejs.org/guide/essentials/composition-api-setup) |
| `context.attrs` | Атрибуты компонента (не-props). | — | <pre><code>setup(props, { attrs }) {<br>  console.log(attrs.class)<br>}</code></pre> | Не реактивный объект, аналог $attrs. Не деструктурировать — использовать attrs.x. Обновляется при обновлении компонента; для side effects — onBeforeUpdate. | [Fallthrough Attributes](https://vuejs.org/guide/components/attrs) |
| `context.slots` | Слоты компонента. | — | <pre><code>setup(props, { slots }) {<br>  console.log(slots.default)<br>}</code></pre> | Не реактивный объект, аналог $slots. Не деструктурировать — использовать slots.x. Свойства slots не реактивны; side effects — onBeforeUpdate. | [Slots](https://vuejs.org/guide/components/slots) |
| `context.emit` | Функция для emit событий. | — | <pre><code>setup(props, { emit }) {<br>  emit('submit', payload)<br>}</code></pre> | Аналог $emit. | [Component Events](https://vuejs.org/guide/components/events) |
| `context.expose()` | Явно ограничивает свойства, доступные родителю через template ref. | — | <pre><code>setup(props, { expose }) {<br>  const publicCount = ref(0)<br>  const privateCount = ref(0)<br>  expose({ count: publicCount })<br>  // expose() без аргументов — ничего не открывает<br>}</code></pre> | expose() без аргументов делает instance «закрытым». expose({ ... }) выборочно открывает refs/methods родителю. | [Composition API — setup()](https://vuejs.org/guide/essentials/composition-api-setup) |
| `setup()` → render function | setup может вернуть render function вместо объекта. | — | <pre><code>import { h, ref } from 'vue'<br><br>export default {<br>  setup(props, { expose }) {<br>    const count = ref(0)<br>    const increment = () => ++count.value<br>    expose({ increment })<br>    return () => h('div', count.value)<br>  }<br>}</code></pre> | Return render function (h/JSX) использует reactive state из setup. При return render function нельзя вернуть объект — для expose методов родителю вызывайте expose() до return. | [Render Functions](https://vuejs.org/guide/extras/render-function) |
