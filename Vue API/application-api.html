Vue Application API

Источник: vuejs.org/api/application

API Описание По умолчанию Пример Детали Ссылки
createApp() Создаёт экземпляр приложения Vue. — 
import { createApp } from 'vue'

const app = createApp({ /* root component options */ })

// или с импортом:
import App from './App.vue'
const app = createApp(App)
 Первый аргумент — корневой компонент, второй (опционально) — props для корневого компонента. Creating a Vue Application
createSSRApp() Создаёт экземпляр приложения в режиме SSR Hydration. — 
import { createSSRApp } from 'vue'
import App from './App.vue'

const app = createSSRApp(App)
 Использование полностью идентично createApp(). SSR — Client Hydration
app.mount() Монтирует экземпляр приложения в DOM-контейнер. — 
import { createApp } from 'vue'
const app = createApp(/* ... */)

app.mount('#app')
// или DOM-элемент:
app.mount(document.body.firstChild)
 Аргумент — DOM-элемент или CSS-селектор. Возвращает экземпляр корневого компонента. mount() можно вызвать только один раз на экземпляр. —
app.unmount() Размонтирует смонтированное приложение. — 
app.unmount()
 Вызывает хуки unmount для всех компонентов в дереве приложения. —
app.onUnmount() Регистрирует callback, вызываемый при размонтировании приложения. — 
app.onUnmount(() => {
  /* cleanup */
})
 — —
app.component() Регистрирует глобальный компонент или возвращает уже зарегистрированный. — 
import { createApp } from 'vue'
const app = createApp({})

app.component('MyComponent', { /* ... */ })
const MyComponent = app.component('MyComponent')
 При передаче только имени — получение компонента. При передаче имени и определения — регистрация. Component Registration
app.directive() Регистрирует глобальную директиву или возвращает уже зарегистрированную. — 
app.directive('myDirective', {
  /* custom directive hooks */
})

app.directive('myDirective', () => { /* ... */ })

const myDirective = app.directive('myDirective')
 Поддерживает объектную директиву и сокращённую функциональную форму. Custom Directives
app.use() Устанавливает плагин. — 
import { createApp } from 'vue'
import MyPlugin from './plugins/MyPlugin'

const app = createApp({ /* ... */ })
app.use(MyPlugin)
 Плагин — объект с методом install() или функция. Повторный вызов с тем же плагином устанавливает его только один раз. Plugins
app.mixin() Применяет глобальный mixin ко всем компонентам приложения. — 
app.mixin({ /* options */ })
 Не рекомендуется. Для переиспользования логики предпочтительны Composables. Composables
app.provide() Предоставляет значение для inject во всех дочерних компонентах приложения. — 
app.provide('message', 'hello')

// Composition API:
import { inject } from 'vue'
setup() { console.log(inject('message')) }

// Options API:
inject: ['message'],
created() { console.log(this.message) }
 Первый аргумент — ключ инъекции, второй — значение. Возвращает экземпляр приложения. Provide / Inject
app.runWithContext() Выполняет callback с текущим приложением как контекстом инъекции. — 
import { inject } from 'vue'

app.provide('id', 1)

const injected = app.runWithContext(() => {
  return inject('id')
})

console.log(injected) // 1
 Требует Vue 3.3+. inject() работает без активного экземпляра компонента. —
app.version Версия Vue, с которой создано приложение. — 
export default {
  install(app) {
    const version = Number(app.version.split('.')[0])
    if (version < 3) {
      console.warn('This plugin requires Vue 3')
    }
  }
}
 Полезно в плагинах для условной логики в зависимости от версии Vue. Global API — version
app.config Объект конфигурации экземпляра приложения. — 
import { createApp } from 'vue'
const app = createApp(/* ... */)
console.log(app.config)
 Свойства следует изменять до монтирования приложения. —
app.config.errorHandler Глобальный обработчик необработанных ошибок приложения. dev: re-throw, prod: log 
app.config.errorHandler = (err, instance, info) => {
  // handle error, e.g. report to a service
}
 Перехватывает ошибки из render, event handlers, lifecycle hooks, setup(), watchers и др. Production Error Code Reference
app.config.warnHandler Пользовательский обработчик runtime-предупреждений Vue. — 
app.config.warnHandler = (msg, instance, trace) => {
  // trace is the component hierarchy trace
}
 Только для отладки. Игнорируется в production. —
app.config.performance Включает трассировку производительности init, compile, render и patch. false 
app.config.performance = true
 Только dev-режим и браузеры с performance.mark. Guide — Performance
app.config.compilerOptions Настройки runtime-компилятора шаблонов. — — Работает только с full build. При runtime-only — через @vue/compiler-dom. compilerOptions
app.config.compilerOptions.isCustomElement Метод проверки, является ли тег нативным custom element. — 
app.config.compilerOptions.isCustomElement = (tag) => {
  return tag.startsWith('ion-')
}
 Возвращает true для нативных custom elements. Vue and Web Components
app.config.compilerOptions.whitespace Настройка обработки пробелов в шаблонах. 'condense' 
app.config.compilerOptions.whitespace = 'preserve'
 'preserve' отключает удаление пробелов между элементами. —
app.config.compilerOptions.delimiters Разделители текстовой интерполяции в шаблоне. ['{{', '}}'] 
app.config.compilerOptions.delimiters = ['${', '}']
 Для избежания конфликтов с mustache-синтаксисом на сервере. —
app.config.compilerOptions.comments Обработка HTML-комментариев в шаблонах. false 
app.config.compilerOptions.comments = true
 true сохраняет комментарии даже в production. —
app.config.globalProperties Объект для регистрации глобальных свойств на экземплярах компонентов. {} 
app.config.globalProperties.msg = 'hello'

// в компоненте:
mounted() { console.log(this.msg) } // 'hello'
 Замена Vue.prototype из Vue 2. Использовать умеренно. Augmenting Global Properties
app.config.optionMergeStrategies Стратегии слияния кастомных опций компонентов. {} 
const app = createApp({
  msg: 'Vue',
  mixins: [{ msg: 'Hello ' }],
  mounted() { console.log(this.$options.msg) }
})

app.config.optionMergeStrategies.msg = (parent, child) => {
  return (parent || '') + (child || '')
}

app.mount('#app') // logs 'Hello Vue'
 Функция получает значения опции родителя и потомка. Component Instance — $options
app.config.idPrefix Префикс для всех ID, генерируемых через useId(). undefined 
app.config.idPrefix = 'myApp'

const id1 = useId() // 'myApp:0'
const id2 = useId() // 'myApp:1'
 — useId()
app.config.throwUnhandledErrorInProduction Принудительно пробрасывает необработанные ошибки в production. false 
app.config.throwUnhandledErrorInProduction = true
 Позволяет сервисам мониторинга перехватывать production-ошибки. —
