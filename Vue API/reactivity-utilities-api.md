# Vue Reactivity API — Utilities

> Источник: [https://vuejs.org/api/reactivity-utilities](https://vuejs.org/api/reactivity-utilities)
>
> **XLSX:** [`reactivity-utilities-api.xlsx`](./reactivity-utilities-api.xlsx) (генерация: `py -3 scripts/generate-reactivity-utilities-api-xlsx.py`)

| API | Описание | По умолчанию | Пример | Детали | Ссылки |
|-----|----------|--------------|--------|--------|--------|
| `isRef()` | Проверяет, является ли значение ref-объектом. | — | <pre><code>import { isRef, ref } from 'vue'<br><br>const foo = ref(1)<br>if (isRef(foo)) {<br>  console.log(foo.value) // 1<br>}</code></pre> | Возвращает true/false. В runtime позволяет безопасно обращаться к .value только у ref. | — |
| `unref()` | Возвращает inner value, если аргумент — ref, иначе сам аргумент. | — | <pre><code>import { unref, ref } from 'vue'<br><br>function useFoo(x) {<br>  const unwrapped = unref(x)<br>  // число, даже если x был ref(число)<br>  console.log(unwrapped)<br>}</code></pre> | Синтаксический сахар: val = isRef(val) ? val.value : val. | — |
| `toRef()` | Нормализует value/ref/getter в ref или создаёт ref для свойства reactive-объекта. | — | <pre><code>import { reactive, toRef, ref } from 'vue'<br><br>// normalization (3.3+):<br>toRef(existingRef)<br>toRef(() => props.foo)<br>toRef(1)<br><br>// свойство reactive-объекта:<br>const state = reactive({ foo: 1, bar: 2 })<br>const fooRef = toRef(state, 'foo')<br>fooRef.value++<br>console.log(state.foo) // 2</code></pre> | Normalization (3.3+): value/ref/getter → ref. Свойство объекта: двусторонний ref, синхронизирован с source. ref(state.foo) не синхронизирован — ref() получает plain value. Для props: toRef(props, 'foo') или toRef(() => props.foo). Мутация ref prop запрещена; для v-model — computed get/set. Работает с optional properties. | [Reactivity Utilities](https://vuejs.org/api/reactivity-utilities#toref) |
| `toValue()` | Нормализует value/ref/getter в plain value. | — | <pre><code>import { ref, toValue, watch } from 'vue'<br><br>toValue(1) // 1<br>toValue(ref(1)) // 1<br>toValue(() => 1) // 1<br><br>function useFeature(id) {<br>  watch(() => toValue(id), (value) => {<br>    console.log(value)<br>  })<br>}<br><br>useFeature(1)<br>useFeature(ref(1))<br>useFeature(() => 1)</code></pre> | Требует Vue 3.3+. Похож на unref(), но также вызывает getter. Удобно в composables, когда аргумент может быть value, ref или функцией-getter. | [Reactivity Utilities](https://vuejs.org/api/reactivity-utilities#tovalue) |
| `toRefs()` | Преобразует reactive-объект в plain object, где каждое свойство — ref. | — | <pre><code>import { reactive, toRefs } from 'vue'<br><br>const state = reactive({ foo: 1, bar: 2 })<br>const stateAsRefs = toRefs(state)<br><br>state.foo++<br>console.log(stateAsRefs.foo.value) // 2<br><br>stateAsRefs.foo.value++<br>console.log(state.foo) // 3</code></pre> | Каждый ref создаётся через toRef(). Полезно при return state из composable — деструктуризация без потери reactivity. Только enumerable свойства; для optional — toRef. | [Reactivity Utilities](https://vuejs.org/api/reactivity-utilities#torefs) |
| `isProxy()` | Проверяет, является ли объект proxy от reactive/readonly/shallowReactive/shallowReadonly. | — | <pre><code>import { reactive, isProxy } from 'vue'<br><br>const state = reactive({ count: 0 })<br>console.log(isProxy(state)) // true<br>console.log(isProxy({ count: 0 })) // false</code></pre> | — | — |
| `isReactive()` | Проверяет, является ли объект proxy от reactive() или shallowReactive(). | — | <pre><code>import { reactive, readonly, isReactive } from 'vue'<br><br>console.log(isReactive(reactive({}))) // true<br>console.log(isReactive(readonly({}))) // false</code></pre> | Не включает readonly proxy. | — |
| `isReadonly()` | Проверяет, является ли значение readonly-объектом. | — | <pre><code>import { readonly, computed, isReadonly } from 'vue'<br><br>console.log(isReadonly(readonly({}))) // true<br>console.log(isReadonly(computed(() => 1))) // true</code></pre> | Свойства readonly могут меняться, но не через прямое присвоение объекту. readonly/shallowReadonly и computed без set считаются readonly. | — |
