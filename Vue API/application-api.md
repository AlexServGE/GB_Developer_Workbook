| API | Описание | По умолчанию | Пример | Детали | Ссылки |
|-----|----------|--------------|--------|--------|--------|
| `createApp()` | Создаёт экземпляр приложения Vue. | — | <pre><code>import { createApp } from 'vue'<br><br>const app = createApp({ /* root component options */ })<br><br>// или с импортом:<br>import App from './App.vue'<br>const app = createApp(App)</code></pre> | Первый аргумент — корневой компонент, второй (опционально) — props для корневого компонента. | [Creating a Vue Application](https://vuejs.org/guide/essentials/application) |
| `createSSRApp()` | Создаёт экземпляр приложения в режиме SSR Hydration. | — | <pre><code>import { createSSRApp } from 'vue'<br>import App from './App.vue'<br><br>const app = createSSRApp(App)</code></pre> | Использование полностью идентично `createApp()`. | [SSR — Client Hydration](https://vuejs.org/guide/scaling-up/ssr#client-hydration) |
| `app.mount()` | Монтирует экземпляр приложения в DOM-контейнер. | — | <pre><code>import { createApp } from 'vue'<br>const app = createApp(/* ... */)<br><br>app.mount('#app')<br>// или DOM-элемент:<br>app.mount(document.body.firstChild)</code></pre> | Аргумент — DOM-элемент или CSS-селектор (используется первый найденный). Возвращает экземпляр корневого компонента. При наличии template/render заменяет содержимое контейнера. В SSR-режиме выполняет гидратацию. `mount()` можно вызвать только один раз на экземпляр. | — |
| `app.unmount()` | Размонтирует смонтированное приложение. | — | <pre><code>app.unmount()</code></pre> | Вызывает хуки unmount для всех компонентов в дереве приложения. | — |
| `app.onUnmount()` | Регистрирует callback, вызываемый при размонтировании приложения. | — | <pre><code>app.onUnmount(() => {<br>  /* cleanup */<br>})</code></pre> | — | — |
| `app.component()` | Регистрирует глобальный компонент или возвращает уже зарегистрированный. | — | <pre><code>import { createApp } from 'vue'<br>const app = createApp({})<br><br>app.component('MyComponent', { /* ... */ })<br>const MyComponent = app.component('MyComponent')</code></pre> | При передаче только имени — получение компонента. При передаче имени и определения — регистрация. | [Component Registration](https://vuejs.org/guide/components/registration) |
| `app.directive()` | Регистрирует глобальную директиву или возвращает уже зарегистрированную. | — | <pre><code>app.directive('myDirective', {<br>  /* custom directive hooks */<br>})<br><br>app.directive('myDirective', () => { /* ... */ })<br><br>const myDirective = app.directive('myDirective')</code></pre> | Поддерживает объектную директиву и сокращённую функциональную форму. | [Custom Directives](https://vuejs.org/guide/reusability/custom-directives) |
| `app.use()` | Устанавливает плагин. | — | <pre><code>import { createApp } from 'vue'<br>import MyPlugin from './plugins/MyPlugin'<br><br>const app = createApp({ /* ... */ })<br>app.use(MyPlugin)</code></pre> | Плагин — объект с методом `install()` или функция, используемая как `install()`. Опции передаются в `install()`. Повторный вызов с тем же плагином устанавливает его только один раз. | [Plugins](https://vuejs.org/guide/reusability/plugins) |
| `app.mixin()` | Применяет глобальный mixin ко всем компонентам приложения. | — | <pre><code>app.mixin({ /* options */ })</code></pre> | **Не рекомендуется.** Поддерживается для обратной совместимости. Для переиспользования логики предпочтительны Composables. | [Composables](https://vuejs.org/guide/reusability/composables) |
| `app.provide()` | Предоставляет значение для inject во всех дочерних компонентах приложения. | — | <pre><code>app.provide('message', 'hello')<br><br>// Composition API:<br>import { inject } from 'vue'<br>setup() { console.log(inject('message')) }<br><br>// Options API:<br>inject: ['message'],<br>created() { console.log(this.message) }</code></pre> | Первый аргумент — ключ инъекции, второй — значение. Возвращает экземпляр приложения. | [Provide / Inject](https://vuejs.org/guide/components/provide-inject), [App-level Provide](https://vuejs.org/guide/components/provide-inject#app-level-provide) |
| `app.runWithContext()` | Выполняет callback с текущим приложением как контекстом инъекции. | — | <pre><code>import { inject } from 'vue'<br><br>app.provide('id', 1)<br><br>const injected = app.runWithContext(() => {<br>  return inject('id')<br>})<br><br>console.log(injected) // 1</code></pre> | Требует Vue 3.3+. Во время синхронного вызова callback `inject()` может получать значения из текущего app даже без активного экземпляра компонента. Возвращает результат callback. | — |
| `app.version` | Версия Vue, с которой создано приложение. | — | <pre><code>export default {<br>  install(app) {<br>    const version = Number(app.version.split('.')[0])<br>    if (version &lt; 3) {<br>      console.warn('This plugin requires Vue 3')<br>    }<br>  }<br>}</code></pre> | Полезно в плагинах для условной логики в зависимости от версии Vue. | [Global API — version](https://vuejs.org/api/general#version) |
| `app.config` | Объект конфигурации экземпляра приложения. | — | <pre><code>import { createApp } from 'vue'<br>const app = createApp(/* ... */)<br>console.log(app.config)</code></pre> | Свойства следует изменять до монтирования приложения. | — |
| `app.config.errorHandler` | Глобальный обработчик необработанных ошибок приложения. | По умолчанию: в dev — re-throw, в prod — log в консоль | <pre><code>app.config.errorHandler = (err, instance, info) => {<br>  // handle error, e.g. report to a service<br>}</code></pre> | Принимает ошибку, экземпляр компонента и строку с информацией об источнике. Перехватывает ошибки из render, event handlers, lifecycle hooks, setup(), watchers, custom directives, transition hooks. В production 3-й аргумент — сокращённый код. | [Production Error Code Reference](https://vuejs.org/error-reference/) |
| `app.config.warnHandler` | Пользовательский обработчик runtime-предупреждений Vue. | — | <pre><code>app.config.warnHandler = (msg, instance, trace) => {<br>  // `trace` is the component hierarchy trace<br>}</code></pre> | Принимает сообщение, экземпляр компонента и trace иерархии. Рекомендуется только для отладки, удалять после. | — |
| `app.config.performance` | Включает трассировку производительности init, compile, render и patch. | `false` | <pre><code>app.config.performance = true</code></pre> | Работает только в dev-режиме и в браузерах с поддержкой `performance.mark` API. | [Guide — Performance](https://vuejs.org/guide/best-practices/performance) |
| `app.config.compilerOptions` | Настройки runtime-компилятора шаблонов. | — | — | Влияет на все компоненты приложения. Можно переопределить на уровне компонента через `compilerOptions`. **Важно:** работает только с full build (`vue.js`). При runtime-only build настройки передаются через `@vue/compiler-dom` в конфигурации сборщика (vue-loader, vite plugin). | [compilerOptions (component)](https://vuejs.org/api/options-rendering#compileroptions) |
| `app.config.compilerOptions.isCustomElement` | Метод проверки, является ли тег нативным custom element. | — | <pre><code>app.config.compilerOptions.isCustomElement = (tag) => {<br>  return tag.startsWith('ion-')<br>}</code></pre> | Возвращает `true`, если тег нужно рендерить как нативный элемент, а не Vue-компонент. HTML и SVG теги распознаются автоматически. | [Vue and Web Components](https://vuejs.org/guide/extras/web-components) |
| `app.config.compilerOptions.whitespace` | Настройка обработки пробелов в шаблонах. | `'condense'` | <pre><code>app.config.compilerOptions.whitespace = 'preserve'</code></pre> | `'condense'`: сжимает пробелы в начале/конце элементов, удаляет пробелы между элементами с переносами строк, сжимает последовательные пробелы в text nodes. `'preserve'` отключает пункты 2 и 3. | — |
| `app.config.compilerOptions.delimiters` | Разделители текстовой интерполяции в шаблоне. | `['{{', '}}']` | <pre><code>app.config.compilerOptions.delimiters = ['${', '}']</code></pre> | Обычно используется для избежания конфликтов с серверными фреймворками, использующими mustache-синтаксис. | — |
| `app.config.compilerOptions.comments` | Обработка HTML-комментариев в шаблонах. | `false` | <pre><code>app.config.compilerOptions.comments = true</code></pre> | По умолчанию Vue удаляет комментарии в production. `true` сохраняет их даже в production. В dev комментарии всегда сохраняются. | — |
| `app.config.globalProperties` | Объект для регистрации глобальных свойств, доступных на любом экземпляре компонента. | `{}` | <pre><code>app.config.globalProperties.msg = 'hello'<br><br>// в компоненте:<br>mounted() { console.log(this.msg) } // 'hello'</code></pre> | Замена `Vue.prototype` из Vue 2. Использовать умеренно. При конфликте с собственным свойством компонента приоритет у свойства компонента. | [Augmenting Global Properties](https://vuejs.org/guide/typescript/options-api#augmenting-global-properties) |
| `app.config.optionMergeStrategies` | Стратегии слияния кастомных опций компонентов. | `{}` | <pre><code>const app = createApp({<br>  msg: 'Vue',<br>  mixins: [{ msg: 'Hello ' }],<br>  mounted() { console.log(this.$options.msg) }<br>})<br><br>app.config.optionMergeStrategies.msg = (parent, child) => {<br>  return (parent || '') + (child || '')<br>}<br><br>app.mount('#app') // logs 'Hello Vue'</code></pre> | Функция стратегии получает значения опции родителя и потомка. Используется плагинами с кастомными опциями через global mixins. | [Component Instance — $options](https://vuejs.org/api/component-instance#options) |
| `app.config.idPrefix` | Префикс для всех ID, генерируемых через `useId()` в приложении. | `undefined` | <pre><code>app.config.idPrefix = 'myApp'<br><br>const id1 = useId() // 'myApp:0'<br>const id2 = useId() // 'myApp:1'</code></pre> | — | [useId()](https://vuejs.org/api/composition-api-helpers#useid) |
| `app.config.throwUnhandledErrorInProduction` | Принудительно пробрасывает необработанные ошибки в production. | `false` | <pre><code>app.config.throwUnhandledErrorInProduction = true</code></pre> | По умолчанию в dev ошибки пробрасываются, в prod — только логируются. `true` позволяет сервисам мониторинга перехватывать production-ошибки. | — |
