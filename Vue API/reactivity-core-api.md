# Vue Reactivity API — Core

> Источник: [https://vuejs.org/api/reactivity-core](https://vuejs.org/api/reactivity-core)
>
> **XLSX:** [`reactivity-core-api.xlsx`](./reactivity-core-api.xlsx) (генерация: `py -3 scripts/generate-reactivity-core-api-xlsx.py`)

| API | Описание | По умолчанию | Пример | Детали | Ссылки |
|-----|----------|--------------|--------|--------|--------|
| `ref()` | Создаёт реактивный и изменяемый ref-объект с единственным свойством `.value`. | — | <pre><code>import { ref } from 'vue'<br><br>const count = ref(0)<br>console.log(count.value) // 0<br><br>count.value = 1<br>console.log(count.value) // 1</code></pre> | Ref изменяемый и реактивный: чтение/запись `.value` отслеживаются. Если в `.value` записан объект — он становится deeply reactive через reactive(). Вложенные refs глубоко разворачиваются. Для shallow — shallowRef(). | [Reactivity Fundamentals](https://vuejs.org/guide/essentials/reactivity-fundamentals) |
| `computed()` | Принимает getter и возвращает readonly computed ref; либо объект `{ get, set }` для writable computed. | — | <pre><code>import { ref, computed } from 'vue'<br><br>const count = ref(1)<br>const plusOne = computed(() => count.value + 1)<br>console.log(plusOne.value) // 2<br><br>const writable = computed({<br>  get: () => count.value + 1,<br>  set: (val) => { count.value = val - 1 }<br>})<br>writable.value = 1<br>console.log(count.value) // 0</code></pre> | Readonly computed нельзя присваивать напрямую. Writable — через get/set. Опционально debuggerOptions: onTrack, onTrigger. | [Computed Properties](https://vuejs.org/guide/essentials/computed) |
| `reactive()` | Возвращает reactive proxy объекта. | — | <pre><code>import { ref, reactive } from 'vue'<br><br>const obj = reactive({ count: 0 })<br>obj.count++<br><br>const count = ref(1)<br>const state = reactive({ count })<br>count.value++ // обновит state.count<br>state.count++ // обновит count.value<br><br>const books = reactive([ref('Vue 3 Guide')])<br>console.log(books[0].value) // .value в массиве</code></pre> | Глубокая реактивность для всех вложенных свойств. Refs внутри объекта разворачиваются, кроме элементов массива/Map/Set — там нужен `.value`. Proxy не равен исходному объекту — работайте с proxy. Для shallow — shallowReactive(). | [Reactivity Fundamentals](https://vuejs.org/guide/essentials/reactivity-fundamentals) |
| `readonly()` | Принимает reactive/plain object или ref и возвращает readonly proxy. | — | <pre><code>import { reactive, readonly, watchEffect } from 'vue'<br><br>const original = reactive({ count: 0 })<br>const copy = readonly(original)<br><br>watchEffect(() => {<br>  console.log(copy.count)<br>})<br><br>original.count++ // ok, watcher сработает<br>copy.count++ // warning, мутация запрещена</code></pre> | Readonly proxy глубокий: вложенные свойства тоже readonly. Ref-unwrapping как у reactive(), но значения тоже readonly. Для shallow — shallowReadonly(). | — |
| `watchEffect()` | Немедленно запускает функцию, отслеживает зависимости и перезапускает при их изменении. | `flush: 'pre'` | <pre><code>import { ref, watchEffect } from 'vue'<br><br>const count = ref(0)<br><br>watchEffect(() => console.log(count.value))<br>// -> 0<br><br>count.value++<br>// -> 1<br><br>const stop = watchEffect(() => {})<br>stop() // остановить watcher</code></pre> | Первый аргумент — effect. Второй — options: flush ('pre' | 'post' | 'sync'), onTrack, onTrigger. Возвращает handle: stop(), pause(), resume(). Cleanup через onCleanup в callback или onWatcherCleanup() (3.5+). flush: 'post' — после render; 'sync' — синхронно (осторожно). | [Watchers](https://vuejs.org/guide/essentials/watchers) |
| `watchPostEffect()` | Псевдоним watchEffect() с `flush: 'post'`. | `flush: 'post'` | <pre><code>import { ref, watchPostEffect } from 'vue'<br><br>const count = ref(0)<br><br>watchPostEffect(() => {<br>  // выполняется после обновления DOM<br>  console.log(count.value)<br>})</code></pre> | Эквивалент watchEffect(fn, { flush: 'post' }). Удобен, когда side effect должен читать обновлённый DOM. | [Callback Flush Timing](https://vuejs.org/guide/essentials/watchers#callback-flush-timing) |
| `watchSyncEffect()` | Псевдоним watchEffect() с `flush: 'sync'`. | `flush: 'sync'` | <pre><code>import { ref, watchSyncEffect } from 'vue'<br><br>const count = ref(0)<br><br>watchSyncEffect(() => {<br>  // синхронно при каждом изменении зависимости<br>  console.log(count.value)<br>})</code></pre> | Эквивалент watchEffect(fn, { flush: 'sync' }). Может влиять на производительность и согласованность данных при пакетных обновлениях — использовать осторожно. | [Callback Flush Timing](https://vuejs.org/guide/essentials/watchers#callback-flush-timing) |
| `watch()` | Отслеживает один или несколько reactive-источников и вызывает callback при изменении. | `immediate: false`, `deep: false`, `flush: 'pre'`, `once: false` | <pre><code>import { ref, reactive, watch } from 'vue'<br><br>const state = reactive({ count: 0 })<br>watch(() => state.count, (count, prevCount) => {<br>  console.log(count, prevCount)<br>})<br><br>const count = ref(0)<br>watch(count, (value, oldValue) => {<br>  console.log(value, oldValue)<br>})<br><br>watch([ref(0), ref('')], ([foo, bar], [prevFoo, prevBar]) => {})</code></pre> | Lazy по умолчанию — callback только при изменении. Источник: getter, ref, reactive object или массив. Options: immediate, deep (или max depth в 3.5+), flush, onTrack/onTrigger, once (3.4+). При прямом watch reactive object — deep mode автоматически. Возвращает handle: stop(), pause(), resume(). Cleanup: onCleanup в callback или onWatcherCleanup(). | [Watchers](https://vuejs.org/guide/essentials/watchers) |
| `onWatcherCleanup()` | Регистрирует cleanup при следующем перезапуске текущего watcher. | — | <pre><code>import { ref, watch, onWatcherCleanup } from 'vue'<br><br>const id = ref(1)<br><br>watch(id, async (newId) => {<br>  const { response, cancel } = doAsyncWork(newId)<br>  onWatcherCleanup(cancel)<br>  data.value = await response<br>})</code></pre> | Только Vue 3.5+. Вызывать синхронно внутри watchEffect или watch callback — не после await в async. Альтернатива onCleanup в watchEffect/watch. failSilently — не бросать ошибку вне watcher. | [Reactivity Core](https://vuejs.org/api/reactivity-core#onwatchercleanup) |
